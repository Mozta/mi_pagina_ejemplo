Workflow colaborativo

Para trabajar en proyectos cuyo desarrollo se realiza en equipo es ideal usar herramientas que permitan facilitar la colaboración. Una de ellas es GitHub que funciona como un repositorio donde se puede llevar a cabo un control de versiones además de poder desarrollar ciertas características del proyecto sin interrumpir el trabajo de los demás, probarlas, corregirlas, etc. manteniendo una organización adecuada. Para tener este orden se trabajan con ramas , creadas preferentemente a partir de develop, para unir todas mediante un merge.

Otra de las herramientas que existe para este tipo de desarrollos es GitKraken, donde podemos visualizar de mejor manera las ramas, archivos locales, trackeados, realizar push, pull, merge y diff para comparar archivos modificados y seleccionar los cambios que queremos conservar en caso de existir conflictos.

API

  • ¿Qué es?

    Es un conjunto de definiciones y protocolos que se utiliza para desarrollar e integrar el software de las aplicaciones.

  • ¿Para qué sirven?

    Permiten que productos y servicios se comuniquen con otros, sin necesidad de saber cómo están implementados.

  • Ventajas

    La mayoría de las APIs ya han sido probadas en todo ambiente de desarrollo, por lo que aseguran un óptimo desempeño en su funcionalidad.

Protocolo de transferencia de hipertexto

  • Es el protocolo de comunicación que permite las transferencias de información a través de archivos en la World Wide Web.

  • Define la sintaxis y la semántica que utilizan los elementos de software de la arquitectura web para comunicarse.
