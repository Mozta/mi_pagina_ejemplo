API

Application Programming Interfaces

¿Qué son las API?

El término API es una abreviatura de Application Programming Interfaces, que en español significa interfaz de programación de aplicaciones. Se trata de un conjunto de definiciones y protocolos que se utiliza para desarrollar e integrar el software de las aplicaciones, permitiendo la comunicación entre dos aplicaciones de software a través de un conjunto de reglas. Las API otorgan flexibilidad; simplifican el diseño, la administración y el uso de las aplicaciones, y proporcionan oportunidades de innovación.

Políticas de versiones de API

Privado: Las API solo se pueden usar internamente, las empresas tienen un mayor control sobre ellas. Esto le da a las empresas un mayor control sobre sus API.

Socios: Las API se comparten con partners empresariales específicos, lo cual puede ofrecer flujos de ingresos adicionales, sin comprometer la calidad.

Público: Todos tienen acceso a las API, esto permite a terceros utilizar éstos para sus proyectos.

Ventajas en el uso de las API

Agilidad o Rapidez: el uso de las Apis de terceros nos permite como desarrolladores integrar en cuestión de minutos funcionalidades como mapas, imágenes o servicios de autenticación sin la necesidad de crear desde cero estas funcionalidades y así disminuir el tiempo de desarrollo.

Garantía de calidad: en gran cantidad de los casos las Apis ya han sido probadas en todos los ambientes de desarrollo por lo que aseguran un óptimo desempeño en su funcionalidad y en cualquiera de los ambientes que se utilice.

Mejora continua: Las grandes compañías no pueden darse el lujo de permitirse ser desplazadas por la competencia así que de manera continua estas se mantienen creando nuevas funcionalidades y mejoras de seguridad a los servicios y funciones que ofrecen mediante el uso de sus Apis.

Si bien el uso de las API puede resultar muy beneficioso, también es importante conocer las desventajas o riesgos que pueden ocurrir con su uso, por ejemplo:

Cambios: Se debe tener en consideración que al estar utilizando servicios de terceros en el desarrollo propio puede llegar a encontrarse en ocasiones que las rutinas para la implementación o utilización de la API se hayan modificado, ya sea por mejora de la misma o por cuestiones de seguridad, y si no se está al tanto de estos cambios se puede llegar a perder el acceso a la misma, también se puede llegar a presentar la situación de que al momento de adquirir el acceso a la API se realizara de manera gratuita y, que un momento determinado, los desarrolladores decidan que este servicio deberá ser de paga.

Control de disponibilidad: Si bien es cierto de que el uso de APIs ahorra tiempo en desarrollo de las aplicaciones o funcionalidades de los sitios web, también es cierto que no deja de ser un servicio de tercero del cual no se puede tener control, si por algún motivo este deja dejara funcionar, no estaría bajo el control del usuario.
